# Security Policy

## Supported version

Security updates are provided for the latest release on the `main` branch.

## Reporting a vulnerability

Do not publish a suspected vulnerability in a public issue. Use GitHub's private vulnerability reporting for this repository. Include the affected version, reproduction steps, impact, and any suggested mitigation.

This application runs entirely in the browser and has no account, server, or production data store. Imported experiment files are size-limited and validated before use, but untrusted files should still be handled cautiously.
