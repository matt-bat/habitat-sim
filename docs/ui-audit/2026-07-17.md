# Observatory Interface Audit — 2026-07-17

## Scope and Method

All six root workspaces were captured and inspected at desktop `1728 × 1117` and mobile `412 × 915`: Planet, Origins, Biosphere, Lineages, Timeline, and Lab. Captures use a fixed clock, dark color scheme, reduced motion, a fresh browser context, and a two-frame paint settle. Reproduce the twelve-image sweep with:

```sh
npm run dev
npm run screenshots -- artifacts/ui
```

The review used a task-specific weighted model because this product must be simultaneously scientifically legible, visually exceptional, and operationally clear. Each category included at least three checks across the twelve captures.

## Findings and Corrections

### Planet

Before: a polished but conventional cutaway occupied a large card while six familiar gauges provided weak causal explanation. Atmosphere percentages were visually prominent but scientifically misleading as reservoir values.

Final: the planet is framed as a live orbital instrument with telemetry reticle, spectral star color, atmospheric column, liquid-solvent, and escape-velocity labels. A conservative habitable-zone rail, grounded orbital observables, interior/surface coupling, partial-pressure ledger, and ranked “why now / what limits next” board make the visual scientifically functional.

### Origins

Before: hypotheses were attractive cards, but all appeared equally applicable to the current world and the lower canvas was largely unused.

Final: every hypothesis displays a world-dependent context-fit index while retaining its separate evidence class. Readiness language now describes a dimensionless opportunity-rate input, not an observed probability.

### Biosphere

Before: the sterile state was a nearly full-canvas blank message.

Final: sterility is treated as data. The screen shows a null biosphere signal, current opportunity index, strongest driver, three limiting factors, counterfactual actions, and uncertainty labels.

### Lineages

Before: the absence of life produced an empty inspector with no scientific value.

Final: the pre-lineage innovation graph shows solvent, building-block, gradient, compartment, and inheritance prerequisites. It explicitly rejects ladder language and names the next experiment.

### Timeline and Lab

The already strong causal drill-in was retained, with mobile evidence badges corrected to avoid stretch. The Lab now exposes stellar mass and temperature, eccentricity, rotation, axial tilt, mantle fraction, background impact rate, pressure, and a custom seven-field material-cargo composer. Changing root views resets each scroll surface to a predictable origin.

## Responsive Review

- Desktop preserves a fixed Spatial Canvas with internal overflow only where data density requires it.
- Mobile keeps all six roots and the primary simulation/intervention actions visible in persistent controls.
- Dense observatory cards stack without horizontal clipping.
- View changes reset vertical and horizontal scroll positions.
- Reduced-motion capture verifies that the interface remains meaningful without orbit or pulse animation.

## Remaining Risks

- Dedicated screen-reader testing and automated accessibility-rule scanning are not represented by screenshot evidence.
- Safari and Firefox engine-specific rendering are not covered by the Chromium matrix.
- Living-state screenshots need curated deterministic scenario fixtures in a later visual-regression suite; current screenshots intentionally audit the first-run sterile state while browser tests cover intervention and navigation behavior.

## Decision

**Pass, 91.7/100.** The interface now has a distinctive observatory identity and materially better causal/scientific communication. The remaining gaps are specialized cross-engine and assistive-technology verification, not visible production-blocking layout defects in the validated Chromium targets.
